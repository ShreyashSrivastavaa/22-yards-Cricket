## Components

### Frontend (Next.js App Router)
- **Purpose:** UI Layer, Client-side analytics, Dashboard management.
- **Location:** `src/`
- **Pattern:** Component-based architecture with Server Components for data fetching.

### Backend (Next.js API Routes)
- **Purpose:** Data aggregation, AI simulation logic, Database interfacing.
- **Location:** `src/app/api/`

### Data Layer
- **PostgreSQL**: Stores normalized match, player, and team data.
- **Redis**: Caches heavy analytical queries (e.g., all-time matchups).

## Data Flow
1. **User requests analytics** (e.g., Virat Kohli vs Left Arm Pace).
2. **Next.js Server Component** checks Redis cache.
3. **If Miss**: Queries PostgreSQL, processes data, updates Redis.
4. **Data returned** and rendered via **Recharts**.
